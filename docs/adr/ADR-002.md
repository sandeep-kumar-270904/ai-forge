# ADR-002: LangGraph for Agent Orchestration

*   **Status:** Accepted
*   **Date:** 2026-06-09

## Context
We need a way to execute multi-step AI agents. Traditional tools like LangChain `AgentExecutor` rely on simple while-loops that are difficult to pause, debug, or serialize. We require the ability to pause an agent mid-thought (for Human-In-The-Loop approvals) and resume it later from an entirely different API pod.

## Decision
We will use **LangGraph**. Workflows will be compiled into stateful graphs where nodes are Python functions (or LLM calls) and edges dictate routing. We will utilize LangGraph's `PostgresSaver` checkpointer to serialize the graph state to the database after every step.

## Consequences
*   **Positive:** We gain "Time-Travel Debugging" (the Replay engine) and secure `interrupt_before` hooks natively out of the box.
*   **Negative:** LangGraph introduces a steep learning curve and requires strict management of the `MessagesState` TypedDict.

## Alternatives Considered
*   **AutoGen / CrewAI:** *Tradeoff:* These frameworks are too "black-box". They do not offer the granular node-level state check-pointing required to pause a process, store it in Postgres, and resume it via an HTTP callback days later.
